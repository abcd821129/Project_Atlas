# AGENTS.md — Codex 工作规则

## 项目

项目名称：Project Atlas / `global-ai-mt5-trader`。

目标是构建全球多资产市场扫描、候选排名、批准策略自主选择、确定性风控和 MT5 模拟账户执行系统。

## 每次开始任务前

1. 阅读 `PROJECT_SPEC.md` 和当前阶段指令。
2. 检查现有代码、配置、测试和 Git 状态。
3. 说明将创建或修改的文件。
4. 只开发用户明确授权的阶段。

## 强制规则

- 不承诺或暗示保证盈利。
- Version 1 禁止真实资金交易。
- `REAL_LIVE` 必须在代码层永久禁用，不能仅靠配置关闭。
- AI只能选择已注册、已测试的策略。
- AI不能决定仓位、修改硬风控、取消止损或提高风险。
- 缺失、过期、冲突、未来时间戳或不合法数据必须导致 `NO_TRADE`。
- 禁止马丁格尔、无限网格、亏损加仓和无止损交易。
- 不得提前开发未授权阶段。
- 不得隐藏异常、删除失败测试或伪造测试结果。
- 不得在代码中写入密钥、账号、密码或真实凭证。
- 不得默认使用付费数据源；必须通过适配器和配置隔离。
- 所有时间戳统一为 UTC，并保留来源时间、接收时间和延迟。
- 所有核心计算必须可复现，避免未来函数和数据泄露。

## 编码标准

- Python 3.12 优先。
- 使用类型标注、Pydantic、结构化日志和小型可维护依赖集。
- 公共接口必须有文档字符串。
- 业务规则放在配置或独立模块，不散落硬编码。
- 测试必须离线可运行。

## 完成任务时

必须报告：

- 创建和修改的文件；
- 架构决策；
- 执行的命令；
- 格式化、类型检查和测试结果；
- 已知限制；
- 下一步建议。

若 Git 已初始化且身份可用，创建一个范围明确的提交。然后停止，不自动进入下一阶段。
