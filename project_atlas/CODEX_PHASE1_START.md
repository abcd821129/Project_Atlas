# Codex 当前任务：Phase 1

请把 `PROJECT_SPEC.md` 作为最高级需求，并遵守 `AGENTS.md`。

当前严格只开发 Phase 1：确定性扫描器。

## 开始前

1. 检查当前仓库、Python环境、已有文件和Git状态。
2. 用中文简要说明现状。
3. 输出Phase 1实施计划。
4. 列出准备创建或修改的文件。
5. 然后直接开始开发。

## 必须完成

- Python包和配置结构；
- 与数据商无关的严格Pydantic模型；
- 仅历史/模拟数据适配器；
- 确定性指标和特征工程；
- 配置化0至100机会评分；
- 按资产类别排名及最终候选限制；
- 相关性和重复风险暴露过滤；
- 确定性市场状态分类；
- 批准策略接口、注册表和 `NO_TRADE`；
- 测试、固定数据、README和本地运行脚本。

## 禁止开发

实时API、新闻、LLM/API调用、Binance WebSocket和订单流、MT5/MQL5、下单、真实资金、复杂UI。

## 工程要求

使用类型标注和Pydantic；计算必须可复现；拒绝缺失、重复、未来、过期和不合法数据；避免未来函数和数据泄露；不得隐藏异常或跳过失败测试；不得虚构密钥或付费服务；依赖应尽量少。

## 完成标准

运行格式化、类型检查和全部测试并修复失败；报告命令和结果；列出变更文件和已知限制；若Git和身份可用，创建一个聚焦提交，否则给出准确提交命令；完成后停止，不进入Phase 2。
